① Action

本质上是一个 JavaScript 对象
包括一个必须的字段 type ，以及其它数据项
类似于这样 {type: ‘Action_NAME’, attr01: ‘data01’, attr02: ‘data02’ }
使用函数 ActionCreator(args) 来创建 Action

②Reducer：状态 state 更新函数

一个纯函数，接收两个参数：state 、action，返回一个新的 state
谨记 reducer 一定要保持纯净
只要传入参数相同，返回计算得到的下一个 state 就一定相同
没有特殊情况、没有副作用，没有 API 请求、没有变量修改，单纯执行计算

①创建 store ，将 reducer 作为参数传入
②使用 store 派发(dispatch) 操作(action)
③store 内部执行 reducer

③Store 状态 state 容器

不仅是 state 容器，更是将 reducer 与 action 连接到了一起
Store 有以下职责
❶维持应用的 state；
❷提供 getState() 方法获取 state；
❸提供 dispatch(action) 方法更新 state；
❹通过 subscribe(listener) 注册监听器;
❺通过 subscribe(listener) 返回的函数注销监听器

谨记:Redux 应用只有一个单一的 store。当需要拆分数据处理逻辑时，你应该使用 reducer 组合而不是创建多个 store

创建

store 的创建很简单，使用 redux 提供的 createStore() ，并传入两个参数，reducer 和 state
reducer : 随着应用的壮大，reducer 也变得复杂，我们需要将其拆分成小的 reducer ，并使用 combineReducers() 合并成一个 reducer
state : 初始状态，可选

Redux 使用的三原则

①Single source of truth单一数据源
整个应用的state，存储在唯一一个object中，同时也只有一个store用于存储这个object

②State is read-only状态是只读的
唯一能改变state的方法，就是触发action操作。action是用来描述正在发生的事件的一个对象

③Changes are made with pure functions
在改变state tree时，用到action，同时也需要编写对应的reducers才能完成state改变操作

React 与 Redux 配合

Redux 并不是 React 的专属，不一定与 React 配合使用
它可以与 React、Angular、Ember、jQuery 甚至纯 javascript 等一起使用
像 React 这种 state => UI 类型的框架与之配合，使用起来还是很爽的

react-redux

使用 react-redux 这个工具把 React 与 Redux 联系起来
提供两个功能： Provider 、 Connect

Provider

Provider : 一个 Component ，用来包裹应用程序的根组件(入口组件)，提供 store 属性，以供子组件使用

Connect

将包装好的组件连接到Redux
尽量只做一个顶层的组件，或者 route 处理
从技术上来说你可以将应用中的任何一个组件 connect() 到 Redux store 中，但尽量避免这么做，因为这个数据流很难追踪
